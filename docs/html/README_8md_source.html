53 ![ccmake screenshot](docs/ccmake_scr.png)

68 ### Configuration Options
69 
70 | Option | Description |
71 |--------------------------------------------------|--------------------------------------------------------------------------------------|
72 |-DALLOC_STATS=ON&#124;OFF | Enable Allocator Statistics |
73 |-DARCH_ALLOC_MAX_SPACE=NUMBER | Set maximum usable allocator space |
74 |-DBREAKS_ENABLE | Enable breakdown reporting |
75 |-DBUILD_TESTS=ON&#124;OFF | Build unit tests |
76 |-DCMAKE_BUILD_TYPE=Debug | Produce debug symbols |
77 |-DCOVERAGE=ON&#124;OFF | Enable gcov coverage |
78 |-DJVineTalk | Build java Vinetalk wrappers |
79 |-DTRACE_ENABLE=ON&#124;OFF | Enable trace file creation |
80 |-DMMAP_POPULATE=ON&#124;OFF | Add MAP_POPULTE flag in mmap. This will make vine_talk_init() slower, use wisely. |
81 |-DUTILS_QUEUE_CAPACITY=NUMBER | Maximum number of outstanding tasks per task queue (Up to 65536), MUST BE power of 2 |
82 |-DVINE_CONFIG_FILE=FILE | Vinetalk configuration file |
83 |-Dasync_architecture=spin&#124;mutex&#124;ivshmem | Method used to ensure ordering |
84 |-Dtarget_architecture=shm | Method used to transfer data |

98 ## Install
99 
100 VineTalk can be 'installed' in two ways.
101 System Wide install is the recomened method if deploying on a dedicated machine and have root/sudo access.
102 User Specific installation is recomended if deploying in a shared machine with multiple users and dont have root/sudo access.
103 
104 ### System Wide Install
105 
106 After a successful build, run <code>make install</code>, with root privileges.
107 
108 ### User Specific Install
109 
110 You can use the LD_LIBRARY_PATH eviroment variable to load VineTalk from the build path.
111 
112 <code>export LD_LIBRARY_PATH=<VineTalk build path></code>
113 
114 To find the apropriate VineTalk build path, run:
115 
116 <code>make VineTalkBuildPath</code>
117 
118 ## Using the Vine Talk Library
119 
120 After a successful build your build directory will have a libvine.so file as well as
121 an include folder. Add your build path as a library path and link with vinetalk <code>-lvine</code>.
122 Also add the build/includes folder to your gcc include paths <code>-Ibuild/includes</code>.

135 ## Breakdowns
136 
137 The BREAKS_ENABLE allow the generation of breakdowns(.brk) and headers (.hdr) for performance evaluation
138 of the VineTalk system.
139 
140 Steps:
141 - Enable BREAKS_ENABLE option and rebuild VineTalk.
142 - Run applications...
143 - When the controller terminates, you will have .hdr and .brk files for all procedures that run.

145 ## Vdf
146 
147 Vdf is a program located at the examples folder, allowing run time inspection of allocator statistics and breakdowns(and more to come).
148 To be built it requires the Poco framework to be installed.
149 
150 After running it spawns a web server at localhost:8888.
151 The web ui allows inspection of allocator statistics and breakdowns.
152 
153 ## shm
154 
155 Shm implements the vinetalk API/protocol over a shared segment
156 (POSIX or ivshmem).
157 
158 ### Required Configuration Keys
159 
160 | Option | Description |
161 |----------|--------------------------------------------------|
162 | shm_file | A file path specifying the shared segments file. |
163 | shm_size | The size of the shared segment in bytes. |
164 
165 ### Optional Configuration Keys
166 
167 | Option | Description |
168 |-------------|-------------------------------------------------------------------------------------------------------------------------|
169 | shm_trunc | A boolean (0,1) setting if the shm_file should be truncated during initialization. |
170 | shm_off | Start mmap from the given byte offset instead from 0.Can be used to split a single shm to multiple vine_pipe instances. |
171 | shm_ivshmem | Boolean , set to 1 if running inside a Vm with ivshmem. |

186 ## VDF

188 VDF is a monitoring tool for Vinetalk, exposing statistics through a web interface.
189 It accepts the following arguements:
190 
191 | Arguement | Description |
192 |--------------------|-----------------------------------------------------------------------|
193 | embed | Skips html and head tags from output, allowing output to be embeded |
194 | noalloc | Dont show allocation statistics |
195 | noobj | Dont show object statistics |
196 | nobreak | Dont show breakdowns |

41 Vinetalk is built as a shared library(libvine.so), using cmake and make:

49 ## Configure with CCMake
50 
51 Run <code>ccmake ..</code> in your build folder and press `c` once:
52 
53 ![ccmake screenshot](docs/ccmake_scr.png)
54 
55 Every line correspond to a build option(see below for option descriptions).
56 To change/set an option press enter, this will toggle a Boolean flag or allow you to edit a string option.
57 For string press enter again to end string input.
58 
59 Once you have configured your build, press `c` followed by `g`.

68 ### Configuration Options
69 
70 | Option | Description |
71 |--------------------------------------------------|--------------------------------------------------------------------------------------|
72 |-DALLOC_STATS=ON&#124;OFF | Enable Allocator Statistics |
73 |-DARCH_ALLOC_MAX_SPACE=NUMBER | Set maximum usable allocator space |
74 |-DBREAKS_ENABLE | Enable breakdown reporting |
75 |-DBUILD_TESTS=ON&#124;OFF | Build unit tests |
76 |-DCMAKE_BUILD_TYPE=Debug | Produce debug symbols |
77 |-DCOVERAGE=ON&#124;OFF | Enable gcov coverage |
78 |-DJVineTalk | Build java Vinetalk wrappers |
79 |-DTRACE_ENABLE=ON&#124;OFF | Enable trace file creation |
80 |-DMMAP_POPULATE=ON&#124;OFF | Add MAP_POPULTE flag in mmap. This will make vine_talk_init() slower, use wisely. |
81 |-DUTILS_QUEUE_CAPACITY=NUMBER | Maximum number of outstanding tasks per task queue (Up to 65536), MUST BE power of 2 |
82 |-DVINE_CONFIG_FILE=FILE | Vinetalk configuration file |
83 |-Dasync_architecture=spin&#124;mutex&#124;ivshmem | Method used to ensure ordering |

98 ## Install
99 
100 VineTalk can be 'installed' in two ways.
101 System Wide install is the recomened method if deploying on a dedicated machine and have root/sudo access.
102 User Specific installation is recomended if deploying in a shared machine with multiple users and dont have root/sudo access.
103 
104 ### System Wide Install
105 
106 After a successful build, run <code>make install</code>, with root privileges.
107 
108 ### User Specific Install
109 
110 You can use the LD_LIBRARY_PATH eviroment variable to load VineTalk from the build path.
111 
112 <code>export LD_LIBRARY_PATH=<VineTalk build path></code>
113 
114 To find the apropriate VineTalk build path, run:
115 
116 <code>make VineTalkBuildPath</code>
117 
118 ## Using the Vine Talk Library
119 
120 After a successful build your build directory will have a libvine.so file as well as
121 an include folder. Add your build path as a library path and link with vinetalk <code>-lvine</code>.
122 Also add the build/includes folder to your gcc include paths <code>-Ibuild/includes</code>.

124 # Configuration
125 
126 In order to configure the vine_pipe endpoints, the user must provide
127 architecture specific options.
128 
129 These configuration options are stored at ~/.vinetalk and follow the format
130 specified in utils/config.h.
131 
132 This is a minimum working configuration:
133 
134 ```
135 shm_file vine_talk
136 ```
137 
138 The sections bellow specify the required keys for each supported vinetalk
139 architecture:
140 
141 ## Breakdowns
142 
143 The BREAKS_ENABLE allow the generation of breakdowns(.brk) and headers (.hdr) for performance evaluation
144 of the VineTalk system.
145 
146 Steps:
147 - Enable BREAKS_ENABLE option and rebuild VineTalk.
148 - Run applications...
149 - When the controller terminates, you will have .hdr and .brk files for all procedures that run.
150 
151 ## Vdf
152 
153 Vdf is a program located at the examples folder, allowing run time inspection of allocator statistics and breakdowns(and more to come).
154 To be built it requires the Poco framework to be installed.
155 
156 After running it spawns a web server at localhost:8888.
157 The web ui allows inspection of allocator statistics and breakdowns.
158 
159 ## shm
160 
161 Shm implements the vinetalk API/protocol over a shared segment
162 (POSIX or ivshmem).
163 
164 ### Required Configuration Keys
165 
166 | Option | Description |
167 |----------|--------------------------------------------------|
168 | shm_file | A file path specifying the shared segments file. |
169 | shm_size | The size of the shared segment in bytes. |
170 
171 ### Optional Configuration Keys
172 
173 | Option | Description |
174 |-------------|-------------------------------------------------------------------------------------------------------------------------|
175 | shm_trunc | A boolean (0,1) setting if the shm_file should be truncated during initialization. |
176 | shm_off | Start mmap from the given byte offset instead from 0.Can be used to split a single shm to multiple vine_pipe instances. |
177 | shm_ivshmem | Boolean , set to 1 if running inside a Vm with ivshmem. |
178
